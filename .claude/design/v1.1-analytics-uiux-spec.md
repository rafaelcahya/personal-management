## 1. VO2max Trend Section ‚Äî Analytics Page (Issue #9)

### Placement

After the Training Load section (`trainingLoadSection`), before Race Predictor (`racePredictorSection`).

### Component structure

Use the existing `Section` wrapper already in `analytics/page.jsx`:

```jsx
<Section id="vo2maxTrendSection" title="VO2max Trend" icon={Activity}>
  ...
</Section>
```

Icon: `Activity` from lucide-react (already imported in analytics page).

### Chart

`LineChart` (recharts) with two data series using the existing `ResponsiveContainer` + `CartesianGrid` + `Tooltip` pattern.

**Data series 1 ‚Äî individual run dots:**

- `dataKey="vo2max"` ‚Äî `estimated_vo2max` per run
- `stroke="#c4b5fd"` (violet-300, lighter)
- `strokeWidth={0}` (no connecting line)
- `dot={{ fill: '#c4b5fd', r: 3 }}`

**Data series 2 ‚Äî 30-day rolling average line:**

- `dataKey="rollingAvg"` ‚Äî computed 30-day window
- `stroke="#7c3aed"` (violet-600)
- `strokeWidth={2}`
- `dot={false}`

**Axes:**

- X-axis: `dataKey="label"`, date formatted `MMM d`, `tick={{ fontSize: 10, fill: '#94a3b8' }}`, `tickLine={false}`
- Y-axis: `domain={['auto', 'auto']}`, `tickFormatter={(v) => v.toFixed(1)}`, unit ` mL/kg/min` in label

**Tooltip:** show date, per-run VO2max (`mL/kg/min` to 1dp), rolling avg value.

**Chart height:** 200px ‚Äî same as other charts on the page.

### Data computation (Frontend responsibility)

```js
// filter to runs with non-null estimated_vo2max, sorted by started_at asc
const pts = activities
  .filter((a) => RUN_TYPES.has(a.activity_type) && a.estimated_vo2max != null)
  .sort((a, b) => new Date(a.started_at) - new Date(b.started_at))
  .slice(-90) // last 90 data points max

// rolling 30-day avg: for each point, avg all pts where started_at is within 30 days prior
```

### States

| State   | Condition                          | UI                                                                                                   |
| ------- | ---------------------------------- | ---------------------------------------------------------------------------------------------------- |
| Empty   | `< 3` runs with `estimated_vo2max` | `EmptyState` component (already in page) ‚Äî copy: `"Connect a HR monitor to unlock VO2max estimates"` |
| Content | `‚â• 3` data points                  | Chart renders                                                                                        |

No separate loading or error state ‚Äî page-level loading/error already handles this.

### Test IDs

| ------------ | -------------------- |

Register in `cypress/fixtures/app-constants.json` and `app-constants.yaml`:

```json

    "vo2maxTrendSection": "vo2maxTrendSection"
  }
}
```

## 2. Efficiency Factor Trend (Issue #10)

### 2a. Activity Detail ‚Äî Stat Tile Trend Arrow

**Location:** `app/main/running/(app)/activities/[id]/page.jsx`
Find the existing EF stat tile at `id="efficiencyFactor_activityDetailPage"`. The EF value is already rendered ‚Äî add the trend arrow inline after the value.

**New field from API:** `efficiency_factor_30d_avg` (float 4dp, nullable) ‚Äî added by Backend #12.

**Arrow logic:**

| Condition                           | Icon           | Class            | `title` attribute                      |
| ----------------------------------- | -------------- | ---------------- | -------------------------------------- |
| `efficiency_factor_30d_avg == null` | none           | ‚Äî                | ‚Äî                                      |
| `efficiency_factor > 30d_avg`       | `TrendingUp`   | `text-green-500` | `"Above your 30-day average (0.0182)"` |
| `efficiency_factor < 30d_avg`       | `TrendingDown` | `text-red-400`   | `"Below your 30-day average (0.0182)"` |
| `efficiency_factor === 30d_avg`     | `Minus`        | `text-slate-400` | `"Equal to your 30-day average"`       |

**Element ID:** `id="efTrendArrow"` on the icon element.

**Placement:** Inline after the EF value number, same line, `size-3.5` icon size to match other inline icons.

**Example JSX:**

```jsx
{
  activity.efficiency_factor != null && activity.efficiency_factor_30d_avg != null && (
    <TrendingUp
      id="efTrendArrow"
      className="size-3.5 text-green-500 inline ml-1"
      title={`Above your 30-day average (${Number(activity.efficiency_factor_30d_avg).toFixed(4)})`}
      aria-hidden="true"
    />
  )
}
```

### 2b. Analytics Page ‚Äî EF Trend Chart Section

**Placement:** Between `vo2maxTrendSection` and `racePredictorSection`.

**Component structure:**

```jsx

  ...
</Section>
```

Icon: `TrendingUp` (already imported in analytics page).

**Chart:** `LineChart` with colored individual dots and a rolling avg line.

**Data series 1 ‚Äî individual colored dots:**

- `dataKey="ef"` ‚Äî `efficiency_factor` per run
- `dot` prop renders green (`#22c55e`) if above rolling avg, red (`#ef4444`) if below
- `strokeWidth={0}` (no line connecting dots)

**Data series 2 ‚Äî rolling avg line:**

- `dataKey="rollingAvg"` ‚Äî 30-day window
- `stroke="#7c3aed"` (violet-600)
- `strokeWidth={2}`
- `dot={false}`
- `strokeDasharray="4 2"` to visually distinguish from individual dot series

**Tooltip:** date + EF value (4dp) + rolling avg (4dp).

**Y-axis:** `tickFormatter={(v) => v.toFixed(3)}`, domain `['auto', 'auto']`.

**Chart height:** 200px.

## 3. Dashboard AI Coach Card (Issue #11)

### What changes

Replace `AiCoachPlaceholder.jsx` with a functional component. The existing placeholder uses `id="aiCoachCard"` ‚Äî keep that ID.

### Card frame

Match the activity detail AI card style (not the plain Card used in placeholder):

```jsx
<section id="aiCoachCard" aria-label="AI Coach">
  <h2 className="text-sm font-semibold text-slate-500 uppercase tracking-wide mb-3">AI Coach</h2>
  <div className="rounded-xl border border-violet-200/60 bg-gradient-to-br from-violet-50 to-purple-50 p-5">
    {/* state content here */}
  </div>
</section>
```

### Data source

`GET /api/running/v1/ai/insights?limit=3` ‚Äî no `activity_id`, no type filter.
Filter client-side to `status === 'completed' && is_valid === true`.

### All states

**Loading** ‚Äî `id="aiCoachLoading_dashboardPage"`

```jsx
<div id="aiCoachLoading_dashboardPage" className="flex flex-col gap-2 animate-pulse">
  <div className="h-3 bg-violet-100 rounded w-3/4" />
  <div className="h-3 bg-violet-100 rounded w-1/2" />
</div>
```

**Empty** ‚Äî `id="aiCoachEmpty_dashboardPage"`

```jsx
<div id="aiCoachEmpty_dashboardPage" className="flex items-start gap-3">
  <div className="rounded-full bg-violet-100 p-2 shrink-0">
    <BrainCircuit className="size-5 text-violet-600" aria-hidden="true" />
  </div>
  <p className="text-sm text-slate-500">
    No AI insights yet ‚Äî complete a run to unlock your first analysis.
  </p>
</div>
```

**Error** ‚Äî `id="aiCoachError_dashboardPage"`

```jsx
<div id="aiCoachError_dashboardPage" className="flex items-center justify-between gap-3">
  <p className="text-sm text-slate-500">Unable to load insights.</p>
  <button
    id="aiCoachRetryBtn_dashboardPage"
    onClick={load}
    className="text-xs text-violet-600 hover:underline shrink-0"
  >
    Try again
  </button>
</div>
```

**Content** ‚Äî `id="aiCoachContent_dashboardPage"`

Show the most recent completed+valid insight:

```jsx
<div id="aiCoachContent_dashboardPage" className="flex flex-col gap-2">
  <p className="text-sm font-semibold text-slate-700 line-clamp-1">{insight.title}</p>
  <p className="text-xs text-slate-500 line-clamp-3">{firstParagraph(insight.content)}</p>
  <div className="flex items-center justify-between mt-1">
    <span className="text-xs text-slate-400">{relativeTime(insight.created_at)}</span>
    {insight.data_refs?.activity_id && (
      <Link
        href={`/main/running/activities/${insight.data_refs.activity_id}`}
        className="text-xs text-violet-600 hover:underline"
      >
        View activity ‚Üí
      </Link>
    )}
  </div>
</div>
```

`firstParagraph(content)` ‚Äî extract text before the first `## ` section heading.
